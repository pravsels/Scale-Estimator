# π0.5 data audit

Primary paper: [π0.5: a Vision-Language-Action Model with Open-World
Generalization](https://arxiv.org/pdf/2504.16054v1).

See the [audit methodology and shared conversion](README.md).

## Model size

- **Paper fact, PDF p. 4, Figure 3:** the pretrained VLM is labelled “SigLIP
  (400M) + Gemma (2.6B)” and the added module “action expert (300M).”
- **Derived:** the nominal final size is
  `400M + 2.6B + 300M = approximately 3.3B`. Component labels are rounded; the
  paper does not provide an exact integer total.

## Included robot source

- **Paper fact, PDF p. 6, §IV-C:** “We use about 400 hours of data of mobile
  manipulators performing household tasks in about 100 different home
  environments…”
- **Paper fact, PDF p. 7, §IV-E:** “We use all four cameras for high-level
  inference, and the wrist and forward cameras for the low-level inference
  process.”
- **Paper fact, PDF p. 7, §IV-E:** “The total dimensionality of the state and
  action spaces is 18 or 19, depending on the platform.”
- **Paper fact, PDF p. 7, §IV-E:** the model commands targets “at 50 Hz (with
  action chunking).”
- **Included as an estimate:** 400 hours using the three low-level views.
- **Assumptions:** 50 Hz control is used as recorded FPS; 224 × 224 input,
  patch size 14, and no spatial pooling are PaliGemma-family proxies not stated
  in this paper; 19 state and 19 action scalar proxies are used; mean episode
  duration is 60 seconds; language is 16 tokens/episode; novelty is 5%.
- **Derived:** `400 × 3,600 × 50 = 72,000,000` assumed control frames and
  `(224 / 14)² × 3 = 768` visual tokens/frame.

## Disclosed but excluded sources

- **Inherited web VLM data — quantity unknown.** PDF p. 4, §IV: “The model
  weights are initialized from a standard VLM trained on data from the web…”
- **Non-mobile robots — quantity unknown.** PDF p. 6, §IV-C: “We also
  collected non-mobile robot data, either with a single arm or two arms, in a
  variety of home environments.”
- **Cross-embodiment/OXE data — quantity unknown.** PDF p. 6, §IV-C: “We also
  include the open-source OXE dataset [15]. This dataset is an extended version
  of the dataset used by π0 [8].”
- **High-level annotations — overlaps robot trajectories.** PDF p. 6, §IV-C:
  “we manually annotate all data with semantic descriptions of the subtasks”
  and “also label relevant bounding boxes.”
- **Web co-training data — consumed quantity unknown.** PDF pp. 6–7, §IV-C:
  “Finally we include a diverse set of web data involving image captioning
  (CapsFusion [87], COCO [12]), question answering (Cambrian-7M [77], PixMo
  [19], VQAv2 [32]), and object localization in pre-training.”
- **Verbal-instruction data — absolute quantity unknown.** PDF p. 11, §V-E:
  “the relatively small verbal instruction dataset… only constitutes about
  11% of the high-level mobile manipulation examples.”
- **Mixture exposure — not unique volume.** PDF p. 2 states: “The overwhelming
  majority of training examples provided to π0.5 (97.6% during the first
  training phase) do not come from mobile manipulators…” PDF p. 7 reports 280K
  pretraining and 80K post-training gradient steps, but batch size is absent.
